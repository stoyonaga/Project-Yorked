Welcome!

Introduction

The goal of this project is to help incoming Computer Science Students at York University find success in their academic careers.

The site aims to accomplish this by providing answers to frequently asked questions, in addition to connecting students to very important resources and contacts. If at any time, you would like to contribute to this project, please reach out to Shogo!

Courses

Each computer science course, divided between the year level gives resources and anecdotal experience.

  1. EECS 1001 - Research Directions in Computing
    • This course introduces students to many computational research avenues. Typically, the Professor will invite many guest speakers.
    • Assessments usually come in the form of quizzes and short essays.
  2. EECS 1012 - Introduction to Computing: A Net-Centric Approach
    • This course introduces students to writing front and back-end web applications. You will learn HTML, CSS, Javascript, Ajax, and PHP.
    • Assessments usually come in the form of assignments, lab tests, midterms, and a final exam.
  3. EECS 1019 - Discrete Mathematics for Computer Science
    • This course introduces students to proofs through cases, counter-example, contradiction, and induction. This is an extremely important and rather intense course. You are highly encouraged to complete all applicable textbook problems in addition to seeking out other forms of practice.
    • Assessments usually come in the form of assignments, midterms, and a final exam.
  4. EECS 1022 - Introduction for Object Oriented Programming
    • This course introduces students to the concept Object Oriented Programming. You will use the Eclipse IDE, and write programs in Java.

  1. EECS 2001 - Introduction to the Theory of Computation
    • This course introduces students to the logic behind computing. You will continue to develop your proofs and theoretical thinking skills.
    • You will learn how to design finite automata and regular expressions.
    • I found this course to be very fun. This course can be rather heavy. You are encouraged to complete all of the textbook readings, homework, and seek out other avenues of practice. You are highly encouraged to attend every tutorial.
    • Assessments usually come in the form of assignments, midterms, and a final exam.
    • While there is no programming in this course, per se, you can test your understanding of the material by implementing concepts. I personally tested my knowledge through Java and Python.
  2. EECS 2011 - Fundamentals of Data Structures
    • This course introduces students to key data structures, sorting algorithms, and mathematical notation behind computing.
    • You will be using Java to complete your programming assignments.
    • It is highly recommended that you practice your understanding of this course by solving LeetCode problems.
    • Assessments usually come in the form of assignments, midterms, and a final exam.
  3. EECS 2021 - Computer Organization
    • This course introduces students to how high-level programming languages are interpreted at the low-level. You will also learn about Computer Hardware and how all the pieces communicate and work together.
    • You will write programs using Assembly (RISC-V), and design circuits through Icarus Verilog.
    • This course has a particularly high workload in comparison to your other second year courses. You are advised to always complete the labs early, and complete all course homework and readings to ensure understanding of the core theory.
  4. EECS 2030 - Advanced Object Oriented Programming
    • This course is an extension of EECS 1022. You will continue to use Java.
    • Rather than just focusing on Object Oriented Programming, you will begin to learn concepts such as Inheritance, Polymorphism, Unit Testing, and Sorting.
  5. MATH 2030 - Elementary Probability
    • Note: This course is only mandatory for the Honours Degree!
    • Assessments usually come in the form of assignments, midterms, and a final exam.
    • It is absolutely integral that you practice every textbook question that you can. This course relies in constant practice to achieve success on the assessments.
    • You should be very familiar with integration from MATH 1310, as it is assumed knowledge in the second half of this course.
  6. EECS 2031 - Software Tools
    • This course introduces students to writing programs in C and Bash.
    • Assessments usually come in the form of assignments, lab tests, midterms, and a final exam.
    • If you have already taken EECS 2011, you can use LeetCode to practice your C skills. Additionally, you can use CodingBat as well.
    • One of the key takeaways from this course is becoming proficient at using Linux. It is recommended that you either ssh into the red servers to practice these concepts, or install WSL2 on your work computer.

  1. EECS 3101 - Design and Analysis of Algorithms
    • This course has a particularly high workload. It is highly encouraged that you complete all of the textbook readings, problems, and seek out additional avenues of practice.
    • In this course, you will learn how to design algorithms, in addition to proving their correctness.
    • This course relies on the completion and understanding of EECS 1019, EECS 2011. It is highly encouraged that you review this material prior to taking the course.
    • LeetCode is very helpful to do while taking this course. It helps you think in terms of designing optimal algorithms, while also using data structures in, "real" applications.
    • Problems on Algorithms (1995) by Ian Parberry is a great resource for learning and practicing this core material.
    • I personally found MIT OpenCourseWare videos for this course to be very helpful.
  2. EECS 3221 - Operating System Fundamentals
    • In this course, you will learn how your Operating System functions, while communicating with hardware.
    • You will explore the concepts of Processes, Threads, Kernel Space, User Space, Process / Memory Algorithms, and Virtual Memory
    • Do note that the assignments in this course are written in C. It is assumed that you are well-versed in this language, in addition to using Linux.
    • You may find this YouTube Playlist by Ghassan Shobaki to be very helpful.
    • As always, you are highly encouraged to read the textbook and complete all applicable homework questions. Reading ahead in this course helps a lot with lecture comprehension.
  3. EECS 3311 - Software Design
    • In this course, students will be introduced to Software Design Patterns. You will also learn about the Software Development Life Cycle!
    • The programming language for this course will depend on the instructor. As of writing this section, the course has been taught using Python, Java, Eiffel, and Fullstack Web Programming.
    • There is no assigned textbook. I highly recommend this YouTube Playlist by Christopher Okhravi.
    • There is a final project for this course. Depending on the instructor, it may be a web application, contributing to an open-source project, or implementing complex algorithms and writing unit tests.
  4. EECS 3421 - Introduction to Database Systems
    • In this course, students will learn how to design a relational database, and implement it using PostgreSQL. You will to write queries and express your thinking through the use of relational algebra.
    • You will have to ssh into the red servers in order to complete and test your assignment solutions.
  5. EECS 3461 - User Interfaces
    • In this course, students will learn how to design a good user interface. You will learn about what factors affect our perception of a good versus bad design decision.
    • You will use Miro and Figma to complete the UI design projects.
  6. EECS 3482 - Introduction to Computer Security
    • In this course, students will learn about the elementary concepts of Computer Security. You will investigate malware, hacking, steganography, and cryptography in depth.
    • The labs for this course don't use a programming language. You will learn to use tools such as Kali Linux and Command Prompt.

  1. EECS 4161 - Mathematics of Cryptography
    • In this course, students will be introduced to many Encryption and Decryption algorithms. Such examples include the Caesar Cipher, Homophonic Substitution Cipher, RSA, and Diffie-Hellman Key Exchange.
    • Programming is required in this course. You are highly encouraged to use SageMath
